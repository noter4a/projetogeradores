 
 
Protocolo de Comunicação Modbus 
K30XL 3.00 / K30XTe 8.10 / Eclipse 2.00 
 
Página  1 
A comunicação dos controladores é baseada no protocolo MODBUS RTU. Esta tabela é compatível 
com os controladores atualizados para comunicação com a plataforma de monitoramento remoto 
KvaNet. As versões de firmware compatíveis são: 
Modelo 
K30XL 3.00 A partir da versão 230202 
K30XTe 8.10 A partir da versão 230125 
Eclipse 2.00 A partir da versão 220829 
 
Especificações 
Protocolo Modbus RTU (Remote Terminal Unit) 
Baud rate 9600 ou 19200 
Paridade Nenhuma, Odd ou Even 
Data bits 8 
Stop bits 1 
Funções 
implementadas 
Read Holding Registers (0x03) 
Write Single Register (0x06) 
Write Multiple Registers (0x10) 
Exception Response (ERROR) 
Interface de 
comunicação RS-232 full duplex 
 
Todos os equipamentos saem de fábrica com a seguinte configuração: 
Configuração de Fábrica 
Endereço 001 (poderá assumir desde o endereço 001 até 247) 
RS-232 19200, 8, N, 1 (19200bps, 8 bits de dados, sem paridade e 1 stop bit) 
 
Observação: Todos os endereços estão em numeração decimal. 
Consulte a disponibilidade do registro de acordo com o modelo do controlador.

5 Indica Aviso WARN

3 Indica que a rede está alimentando a carga RAC 
2 Indica que o controlador está no modo Inibido INI 
1 Indica que o controlador está no modo Manual MAN
